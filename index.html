django

web项目：前端、后端

前端：html css js

后端（后台）：java/python mybatis,django:数据传递和存储（存储在数据库）

安装django:pip install django

创建项目：django-admin startproject 项目名

创建应用：先进入manage.py文件的同层文件夹中，命令：python manage.py startapp 应用名

运行项目：

# 1、打开manage.py文件
# 2、右键运行
# 3、配置参数：name：项目名称，parameter:runserver （端口号）

配置项目的应用

#1、在settings.py的INSTALLED_APPS中：注册应用,"demo"  (demo是应用名)
#2、总路由分发：urls.py :
            # 导入 from django.urls import include
            # path('demo/',include("demo.urls"))
#3、在应用中创建子路由urls.py，
            # 导入视图类：from . import views
#4、在应用的views.py中创建视图函数（网页）
            #导入 from django.http import HttpResponse
            # 定义视图函数
                #def index(request):
                    #return HttpResponse("视图内容")
#5、在子路由urls.py文件中添加视图路由
            # path('index',views.index) index:视图路由，views.index:视图函数

我的博客

MQ（message queue）分布式系统中的重要组件
解决应用解耦，消息异步分发，流量削峰，消息通讯
MQ产品：RabbitMQ,RocketMQ,Kafka,ActiveMQ,ZeroMQ
特点：消费-生产模式的一个典型代表，MQ和JMS类似，JMS是sun java消息中间件服务的一个标准和APi定义MQ是遵循AMQP协议的具体体现和产品
JMS用于提供消息服务的技术规范，MQ是面向消息中间件的最终实现
优势：MQ是异步的，往对罗列里发送消息无需等待，不同与通信协议
JMS（java messaging service）简介：java消息服务是java平台关面向消息中间件的技术规范，通过提供标准的产生，发送，接收消息的接口简化企业应用开发
JMS术语：
Producer/Provider:生产者
Consumer:消费者
Publisher:发布者
Subscriber:订阅者
moder:消息模型，在客户端之间传递消息的方式
PTP：point to point,点对点的消息模型(一个消息只有一个人可以收到)
Pub/Sub:发布/订阅的消息模型（一个消息可以有多个消费者）
Queue:队列目标
Topic:主题目标
ConnectionFactory:连接工厂，JMS用他创建连接
MQ术语：
队列管理器：为我们提供基于队列的消息服务
消息：1，消息描述符（消息的优先级，生命周期，消息id）2消息体（非永久性消息，永久性消息）
队列：消息的存放地
MQ中队列类型：
本地队列，远程队列，模板队列，动态队列等
通道：channel(d队列管理器之间传递消息的管道，MQ产品的精华)
